    函数式编程——闭包

    vasonYin 2021-3-15 JavaScript

    闭包 (Closure)：有权访问另一个函数作用域中的变量的函数。

    # 闭包产生原因

    函数在执行的时候会放到一个执行栈上当函数执行完毕之后会从执行栈上移除，但是堆上的作用域成员因为被外部引用不能释放，因此内部函数依然可以访问外部函数的成员。

    • 函数和其周围的状态(词法环境)的引用捆绑在一起形成闭包。

    • 可以在另一个作用域中调用一个函数的内部函数并访问到该函数的作用域中的成员

    # 闭包的作用

    # 提升变量的生命周期

    变量的生命周期：从变量的声明到变量的回收。

    • 局部变量：从声明这个变量到这个函数执行结束
    • 全局变量：js 代码从开始执行到执行结束
    function fn() {
  let num = 10
  function add() {
    console.log(num)
  }
  function rem(str) {
    let str1 = str > 0 ? str : 0
  }
  return {
    getNum: add,

}

    # 声明私有属性

    function Student(name) {
  let age = 20
  this.name = name
  this.setAge = function (value) {
    if (value >= 0 && value <= 100) {
      age = value
      console.log(age)
    }
    this.getAge = function () {
      return '我的年龄不方便透露....'
    }
  }
}
let s2 = new Student('luck')
console.log(s2.name) //luck
s2.setAge(30)
